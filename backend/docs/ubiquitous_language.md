# Ubiquitous Language

This document defines the core terms and domain language used throughout the Multi-Asset Autonomous Paywall Optimizer (APO) project.

## Domain Terms

### Paywall
The user interface screen displayed to mobile application users prompting them to purchase a subscription or unlock premium features.

### Paywall Mutation
A specific configuration change applied to a paywall, including parameters such as price point, localized currency, UI color theme, layout styles, and Call-to-Action (CTA) copywriting.

### Cohort
A group of users sharing common characteristics (e.g., install date, application property, user behavior) tracked collectively for analytical comparisons.

### A/B Isolation Test
A controlled experiment where user cohorts are split into distinct treatment paths (e.g., 90% Control receiving the original paywall, and 10% Test receiving a mutated paywall layout) to evaluate performance differences.

### Segment / Segmentation
The deterministic grouping of users into test or control variants based on stable identifiers (e.g., hashing user UUIDs) to ensure they receive a consistent experience across sessions.

### Conversion Rate (CR)
The ratio of active users who complete a purchase transaction on a paywall relative to the total number of users who viewed (impressioned) the paywall.

### Lifetime Value (LTV)
The projected revenue a single user generates across their entire lifecycle within the application ecosystem.

### Telemetry Stream
A continuous stream of real-time user action events (e.g., impressions, clicks, purchases) generated by mobile applications and ingested by the backend.

### Tumbling Window
A fixed-size, non-overlapping, and contiguous time window (e.g., 5 seconds) used to aggregate telemetry metrics dynamically.

### Human-in-the-Loop (HITL)
An operational model where autonomous AI optimization suggestions (e.g., layout mutations) are presented to a human manager for manual approval/slider isolation before live production deployment.

### Vector Embeddings
A numerical representation of textual features and cohort metrics generated by an embedding model (e.g., `all-minilm`), mapping semantically similar historical failures and mutation events close together in a vector space.

### Retrieval-Augmented Generation (RAG)
An architectural pattern where relevant historical paywall mutations and their corresponding conversion rates are retrieved from a vector database (using pgvector) to ground and guide the LLM (e.g., Qwen2.5) in synthesizing a contextually optimized paywall remediation variant.
